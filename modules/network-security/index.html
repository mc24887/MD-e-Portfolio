Network Security Completed

Module Overview

Segmentation, access control and protocol security. Packet analysis surfaced misconfigurations and clear-text patterns; implemented least-privilege network design and TLS hygiene across services.

Key Submissions

  • Packet analysis brief — Identified unauthenticated services, weak crypto and unnecessary exposure using Nmap, OpenVAS, ZAP and Wireshark; delivered fixes and retests.
  • Segmentation rationale — Tiered architecture and inter-tier ACLs; micro-segmented high-value assets; kept low-risk areas simple to reduce ops toil.
Recon & enumeration (commands & excerpts) 
# Baseline service & OS detection
nmap -sC -sV -O -Pn target.example.com
# TLS protocols & cipher suites
nmap -sV -p 443,8443 --script ssl-enum-ciphers target.example.com

Findings: HTTP on 8080 (no TLS); TLSv1.0 enabled; verbose SSH banners. Actions: force TLS-only, enable HSTS, remove weak suites, minimise banners.

Web hardening (ZAP headers) 
X-Frame-Options: DENY
Content-Security-Policy: default-src 'self'
Referrer-Policy: no-referrer
X-Content-Type-Options: nosniff
Strict-Transport-Security: max-age=31536000; includeSubDomains; preload
Packet-level verification (Wireshark/tshark) 
# Pre-fix: clear-text POSTs
tshark -Y "tcp.port == 8080 and http.request.method == \"POST\"" -T fields -e ip.src -e http.host -e http.request.uri
# Post-fix: TLS floor & no secrets
tshark -Y "tls.handshake.version" -T fields -e frame.number -e tls.handshake.version
Segmentation & ACL examples 
ip access-list extended APP-IN
  remark Allow HTTPS from WAF only
  permit tcp host 10.0.10.5 10.0.20.0 0.0.0.255 eq 443
  deny   ip 10.0.0.0 0.0.255.255 10.0.0.0 0.0.255.255
  permit ip any any log
ufw default deny incoming
ufw allow from 10.0.10.5 to any port 443 proto tcp comment "WAF → app only"
