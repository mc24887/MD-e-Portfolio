Network Security Completed

Module Overview

Segmentation, access control and protocol security. Packet analysis surfaced misconfigurations and clear-text patterns; implemented least-privilege network design and TLS hygiene.

Key Submissions

  • Packet analysis brief — Documented unauthenticated services, weak crypto and unnecessary exposure using captures; proposed scoped services, TLS policy and credential hygiene.
  • Segmentation rationale — Defined tiers and inter-tier rules; justified micro-segmentation for high-value assets; retained simplicity elsewhere.
Recon & enumeration (commands & excerpts) 
# Baseline service & OS detection
nmap -sC -sV -O -Pn target.example.com

# TLS protocols & cipher suites (identify legacy/weak)
nmap -sV -p 443,8443 --script ssl-enum-ciphers target.example.com

# Top UDP for exposed infra (noise-controlled)
nmap -sU --top-ports 20 --defeat-rst-ratelimit target.example.com

Findings (representative): HTTP on 8080 (no TLS); TLSv1.0 enabled on 443 with weak ciphers; verbose banners on SSH. Actions: enforce TLS-only, enable HSTS, drop TLSv1.0/1.1 and weak suites, minimise banner leakage.

Web application hardening (ZAP headers) 
X-Frame-Options: DENY
Content-Security-Policy: default-src 'self'
Referrer-Policy: no-referrer
X-Content-Type-Options: nosniff
Strict-Transport-Security: max-age=31536000; includeSubDomains; preload

Post-fix scan showed reduced risk surface (clickjacking, MIME sniffing) and enforced HTTPS across sub-domains.

Packet-level verification (Wireshark/tshark) 
# Locate clear-text credentials pre-fix
tshark -Y "tcp.port == 8080 and http.request.method == \"POST\"" -T fields -e ip.src -e http.host -e http.request.uri

# Confirm protocol floor post-fix
tshark -Y "tls.handshake.version" -T fields -e frame.number -e tls.handshake.version

# Quick check for secrets in the clear
tshark -Y "http && (authorization || (form-urlencoded && http.request.method == \"POST\"))"
  • Before: plaintext POST credentials on 8080; mixed TLS versions on 443.
  • After: TLS-only with HSTS; no clear-text secrets observed; TLS≥1.2 only.
Segmentation & ACL examples 
# Router ACL (edge → app-tier)
ip access-list extended APP-IN
  remark Allow HTTPS from WAF only
  permit tcp host 10.0.10.5 10.0.20.0 0.0.0.255 eq 443
  remark Deny lateral movement by default
  deny   ip 10.0.0.0 0.0.255.255 10.0.0.0 0.0.255.255
  permit ip any any log
# Linux host firewall (least-privilege)
ufw default deny incoming
ufw default allow outgoing
ufw allow from 10.0.10.5 to any port 443 proto tcp comment "WAF → app only"

Rules applied at multiple control points (edge, distribution, host) with logging to validate policy intent.

TLS policy (minimums & hygiene) 
# Example nginx snippet (protocol/cipher floor)
ssl_protocols TLSv1.2 TLSv1.3;
ssl_ciphers   HIGH:!aNULL:!MD5:!3DES:!RC4;
ssl_prefer_server_ciphers on;
ssl_session_tickets off;

Certificate management improved via short-lived certs, OCSP stapling and automated rotation.
