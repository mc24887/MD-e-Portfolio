Module Overview

Built a practical assessment pipeline across host and web layers using Nmap, OpenVAS, OWASP ZAP and Wireshark. Automated results were validated at packet level and turned into an impact‑driven remediation plan.

Key Submissions & Evidence

Service & TLS enumeration (Nmap)

nmap -sC -sV -O -Pn target.example.com
nmap -sV -p 80,443,8080 --script ssl-enum-ciphers target.example.com
  • Found HTTP on 8080 (plaintext) and weak TLS suites on 443.
  • Actions: force TLS‑only, enable HSTS, remove legacy ciphers, minimise banner leakage.

Vulnerability scanning & triage (OpenVAS)

  • Prioritised items with clear exploit paths (credential exposure, weak TLS) over low‑impact noise.

Web assessment (OWASP ZAP)

# Example hardening
X-Frame-Options: DENY
Content-Security-Policy: default-src 'self'
Referrer-Policy: no-referrer
X-Content-Type-Options: nosniff
Strict-Transport-Security: max-age=31536000; includeSubDomains; preload

Packet‑level verification (Wireshark)

tcp.port == 80 and http
http.request.method == "POST"
tls.handshake.version
  • Before: plaintext POST credentials on 8080.
  • After: TLS‑only with HSTS; no sensitive content observed in clear‑text.

Supporting Artefacts

Reflection

Combining automation with packet‑level proof reduced false positives and produced remediation that operations could apply quickly.
