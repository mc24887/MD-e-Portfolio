Secure System Architecture Completed

Module Overview

Reference architecture with explicit trust boundaries, zero-trust communication and layered controls. Prioritised weaknesses using CWE/CWSS and validated choices via executable checks and logs.

Key Submissions

  • CWSS prioritisation — Input validation and authn/authz defects ranked highest due to frequency and exploitability.
  • Attack/Defence Trees — Multi-stage paths and countermeasures mapped to decision points.
  • Simulation — Sanity checks for mTLS, signed messages, origin verification and role-based access.
